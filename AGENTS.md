# UrbanFresh agent instructions

## Start every chat here

Before inspecting or changing this repository, read `CHANGELOG.md` completely. Treat its **Current state**, **Open items**, and newest dated entry as the project handoff from the previous chat.

## Keep the handoff current

Update `CHANGELOG.md` whenever a material task is completed, blocked, deployed, scheduled, or externally verified. Make the update in the same change as the work it describes.

- Keep entries newest first and use `YYYY-MM-DD` dates.
- Record the outcome, important files, validation, commit/PR/deployment status, and any remaining next action.
- Update **Current state** when the live or automation state changes.
- Add unresolved warnings to **Open items** and remove them only after verification.
- Preserve concise, factual handoff information that lets a new chat continue without reconstructing history.
- Never record passwords, private keys, service-account JSON, OAuth tokens, `.env.local` contents, or other secrets.
- Do not claim a deployment, Search Console submission, or fix succeeded unless it was actually verified.

Existing unrelated worktree changes belong to the user. Do not stage, overwrite, or include them merely to update the changelog.
